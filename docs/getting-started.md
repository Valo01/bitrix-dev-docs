# Старт разработки

Что сделать в начале работы с любым Bitrix-проектом команды.

## 1. Окружение

### Что сделать на старте

1. **Поднять окружение** — OSPanel, Docker, VMBitrix или аналог: PHP 8.2+, MySQL 8.0+, веб-сервер.
2. **Создать домен** — пустая папка, которая станет document root сайта.
3. **Установить ядро Bitrix** — через [BitrixSetup](#установка-1с-битрикс-управление-сайтом-через-bitrixsetupphp) (см. ниже).
4. **Проверить** — главная открывается, вход в `/bitrix/admin/` работает.
5. **Настроить конфигурацию проекта** — вынести параметры, которые зависят от стенда (ID инфоблоков, форм, API-ключи), в принятый в проекте способ: `.env`, `constants.php`, хелпер-класс — см. [configuration.md](configuration.md). Числа не пишут напрямую в `index.php` и шаблонах.

#### Быстрая проверка готовности

| Проверка | Ожидаемый результат |
|----------|---------------------|
| Главная страница | Открывается без PHP-ошибок |
| `/bitrix/admin/` | Форма входа или админка |
| Конфиг проекта | ID и ключи/пароли от внешних сервисов заполнены (`.env`, `constants.php`) |

Если что-то из этого не выполняется — не переходите к задачам, сначала закройте пункты выше.

### Установка «1С-Битрикс: Управление сайтом» через BitrixSetup.php

Скрипт [BitrixSetup](https://www.1c-bitrix.ru/download/cms.php#tab-subsection-2) скачивает дистрибутив продукта **напрямую на сервер** и распаковывает его через браузер — без ручной загрузки `.zip` на компьютер и без SSH.

**Когда нужен:** новый локальный стенд, тестовый сайт, демо на 30 дней (потом можно активировать лицензию и сохранить данные).

#### Требования

По [официальной странице загрузки](https://www.1c-bitrix.ru/download/cms.php#tab-subsection-2):

- **PHP 8.2** и выше
- **Apache** 2.0+
- **MySQL** 8.0+

#### Шаги (OSPanel + BitrixSetup)

1. **Создайте домен** в OSPanel, например `C:\OSPanel\domains\my-bitrix\` — папка должна быть document root сайта.
2. **Скачайте** [BitrixSetup.php](https://www.1c-bitrix.ru/download/cms.php#tab-subsection-2) с официального сайта.
3. **Положите** файл в **корень** домена (рядом с будущими `bitrix/`, `index.php`):

```
C:\OSPanel\domains\my-bitrix\
└── bitrixsetup.php
```

4. **Запустите** OSPanel (Apache/Nginx + MySQL + нужная версия PHP).
5. **Откройте в браузере:** `http://my-bitrix/` или `http://my-bitrix/bitrixsetup.php`.
6. **Следуйте мастеру установки:**
   - выбор редакции (Старт, Стандарт, Малый бизнес и т.д.);
   - проверка прав на запись в каталог;
   - загрузка и распаковка дистрибутива;
   - создание БД (имя, пользователь, пароль MySQL из OSPanel);
   - настройка администратора сайта.
7. **После установки** удалите или переименуйте `bitrixsetup.php` из корня (рекомендация безопасности).
8. Проверьте: главная открывается, вход в `/bitrix/admin/` работает.

#### После установки ядра

Ядро `bitrix/` в репозиторий **не** коммитят — его ставят установщиком или обновляют через админку. Кастомный код проекта — в `local/` и при необходимости в `/include/`.

## 2. Первые файлы для изучения

Изучите их **в вашем проекте** — названия и содержимое могут отличаться:

| # | Файл | Зачем |
|---|------|-------|
| 1 | Конфиг ID (`.env`, `constants.php`) | ID инфоблоков, форм, API-ключи |
| 2 | `local/php_interface/init.php` | Автозагрузка, события, хелперы |
| 3 | `local/templates/<шаблон>/header.php` | Шапка, глобальные компоненты |
| 4 | `local/templates/<шаблон>/footer.php` | Подвал, формы, скрипты |
| 5 | `index.php` (главная) | Типовое подключение компонентов |
| 6 | `include/` или `local/templates/.../include/` | Включаемые области |
| 7 | `urlrewrite.php` | Правила ЧПУ |

## 3. Шаблон сайта

Шаблон сайта — общая обёртка всех страниц: `header.php`, `footer.php`, стили и переопределения стандартных компонентов.

**Активный шаблон** выбирается в админке: **Настройки → Настройки продукта → Сайты** → редактирование сайта → поле «Шаблон сайта».

**Файлы шаблона** лежат в:

```
local/templates/<имя_шаблона>/
├── header.php
├── footer.php
├── description.php
└── components/
    └── bitrix/
        ├── news.list/<шаблон>/
        ├── news/<шаблон>/
        ├── catalog/<шаблон>/
        ├── menu/
        └── form.result.new/
```

Имя шаблона в проектах часто `main`.

### Как создать шаблон

На старте шаблон обычно уже есть в `bitrix/templates/` — его ставит установщик или мастер сайта. Дальше работают так: **сначала назначают шаблон сайту в админке, потом переносят копию в `local/`** и правят только там.

#### 1. Назначить шаблон сайту

**Настройки → Настройки продукта → Сайты** → открыть нужный сайт → поле **«Шаблон сайта»** → выбрать шаблон (в проектах часто `main` или тот, что создал мастер установки) → **Сохранить**.

Откройте главную в браузере и убедитесь, что сайт отображается с выбранным шаблоном.

На этом шаге Bitrix берёт файлы из `bitrix/templates/<имя_шаблона>/`.

#### 2. Перенести шаблон в `local/`

Скопируйте **всю папку** шаблона:

```
bitrix/templates/<имя_шаблона>/   →   local/templates/<имя_шаблона>/
```

Пример для шаблона `main`:

```
bitrix/templates/main/   →   local/templates/main/
```

**Имя папки должно совпадать** с тем, что выбрано в админке на шаге 1. Менять его не нужно.

После копирования Bitrix автоматически использует версию из `local/templates/` — она имеет приоритет над `bitrix/templates/`.

#### 3. Править только `local/`

Дальнейшие изменения — только в `local/templates/<имя_шаблона>/`:

- `header.php`, `footer.php` — шапка и подвал;
- `components/bitrix/...` — переопределения компонентов по мере задач.

Папку `bitrix/templates/<имя_шаблона>/` **не редактируют** — при обновлении ядра правки там потеряются.

#### Переопределения компонентов — по мере задач

Когда нужна своя вёрстка списка, меню или формы — копируют шаблон компонента из ядра в:

```
local/templates/<имя_шаблона>/components/bitrix/<компонент>/<шаблон>/
```

Приоритет поиска и примеры путей — в [structure.md](structure.md). Подключение в шапке и подвале — в [components/header-footer.md](components/header-footer.md).

## 4. ID инфоблоков — не в коде страниц

Допустимые способы (зависит от проекта):

```php
// Вариант 1: .env + Dotenv
'IBLOCK_ID' => $_ENV['IBLOCK_ID_NEWS'],

// Вариант 2: константы в init.php / constants.php
'IBLOCK_ID' => IBLOCK_NEWS_ID,

// Вариант 3: хелпер-класс
'IBLOCK_ID' => \Project\IBlock::get('news'),
```

**Неправильно:** `'IBLOCK_ID' => 7` прямо в `index.php`.

Подробнее — [configuration.md](configuration.md).

## 5. Настройки сайта

Телефон, почта, логотипы обычно хранятся в модуле настроек или UF-полях и читаются через `COption`:

```php
\COption::GetOptionString('askaron.settings', 'UF_PHONE');
\CFile::GetPath(\COption::GetOptionString('askaron.settings', 'UF_LOGO'));
```

Конкретный модуль - `askaron.settings`.

## 6. Включаемые области

Два распространённых варианта размещения:

| Вариант | Путь |
|---------|------|
| В корне сайта | `/include/<раздел>/file.php` |
| В шаблоне | `/local/templates/<шаблон>/include/<раздел>/file.php` |

Подключение одинаковое:

```php
$APPLICATION->IncludeComponent('bitrix:main.include', '', [
    'AREA_FILE_SHOW' => 'file',
    'PATH' => '/include/about/banner.php',  // или путь в шаблоне
]);
```

Подробнее — [include-areas.md](include-areas.md).

## 7. Типовая страница

```php
<?php
require $_SERVER['DOCUMENT_ROOT'] . '/bitrix/header.php';

$APPLICATION->SetTitle('Заголовок');
$APPLICATION->SetPageProperty('description', '...');
?>

<?php
// Хлебные крошки — на внутренних страницах, не на главной
$APPLICATION->IncludeComponent('bitrix:breadcrumb', '', [
    'START_FROM' => '0',
    'SITE_ID' => SITE_ID,
]);
?>

<!-- main.include, news.list, bitrix:news / bitrix:catalog -->

<?php require $_SERVER['DOCUMENT_ROOT'] . '/bitrix/footer.php'; ?>
```

В некоторых проектах крошки вынесены в хелпер `ShowNavChain()` в `init.php`.
